Post-mortem Report for TAME project

What where the challenges?

  • I was inexperienced in:
    • Python
    • Machine learning
    • Neural networks
    • Pycharm
    • Pytorch
    • Researching an academic topic through papers
    • Working
    • Juggling work and university
    • Using linux tools (bash, command line tools etc.)
    • Writing an academic paper to be published
  • I procrastinated a lot of my time being non-productive.
  • I had no timetable to adhere to.
  • I did not know if I would be offered a chance to continue working.

What went well?

  • I was able to learn a lot of things, such as:
    • Python
    • Neural networks
    • Pytorch
    • Reading and understanding papers
    • Stress management
    • Writing academic papers
  • I took a chill approach to challenges by methodically learning new skills, studying the docs and trying things out. I think this should be something to always do when time isnâ€™t extremely constrained

Were there issues with the quality or compromises along the way?

Yes:

  • I did not try all the possibilities I wanted to. The training took way too long so a lot of things dragged out much more than anticipated.
  • I started writing the paper just 1.5 weeks before the deadline for submissions, so the whole process felt rushed and grueling. I was not totally satisfied with my work and felt like I should have started writing the paper much sooner, as to be able to supplement it with extra experimental data.

What would you change or improve for future projects?

  • I would be much more careful with my time management, by making an anticipated and periodically updating it to understand which parts are taking too long and should be hurried.
  • I would do a more general overview of the research area and outline several possibilities at the start of the project, instead of constantly trying out possibilies (which is not bad but should not be overdone imo).
